## <span style="color:#0D47A1;">Exercise 1: Initialize the Project & Set Up copilot-instructions.md</span>

Create a fresh React + TypeScript project and use Copilot Agent mode to bootstrap it with your team's standards file.

### <span style="color:#0D47A1;">Step 1.1: Create Your Project with Vite</span>

```bash
# Create the React + TypeScript project
npm create vite@latest my-form-library -- --template react-ts
cd my-form-library

# Initialize git
git init
echo "node_modules/\ndist/\ncoverage/" >> .gitignore
git add . && git commit -m "Initial Vite + React TS scaffold"

# Open in VS Code
code .
```

### <span style="color:#0D47A1;">Step 1.2: Ask Copilot Agent to Generate copilot-instructions.md</span>

Open Copilot Chat and switch to **Agent mode**. Then type:

```text
@agent This is a new React + TypeScript component library project.
Review the existing files (#file:package.json #file:tsconfig.json)
and generate .github/copilot-instructions.md with:
- Project overview and purpose
- TypeScript conventions (strict mode, no implicit any)
- React component conventions (function components, named exports)
- Testing conventions (React Testing Library, jest-dom matchers)
- Import style (absolute paths via tsconfig paths)
- File naming conventions (PascalCase for components, camelCase for hooks)
```

Agent will create the `.github/` directory and the file. Open it and review what was generated.

## <span style="color:#0D47A1;">Exercise 2: Customize copilot-instructions.md for Your Team</span>

Transform the generated draft into a complete project standards file. This is where you invest time upfront to save time on every future Copilot interaction.

### <span style="color:#0D47A1;">Step 2.1: Ask Copilot to Expand the Standards File</span>

With `#file:.github/copilot-instructions.md` in context, type:

```text
Update #file:.github/copilot-instructions.md to add these standards:

Component patterns:
- All components accept a className prop for style overrides
- All interactive components forward refs using React.forwardRef
- All props interfaces are named [ComponentName]Props and exported

Accessibility:
- All form inputs must have associated <label> elements
- All interactive elements must have accessible names (aria-label or
  visible text)
- Color contrast must meet WCAG 2.1 AA (4.5:1 for text)

Testing:
- Tests use @testing-library/react and @testing-library/user-event
- Test files live next to their component: ComponentName.test.tsx
- Never test implementation details — test user-visible behavior

Forbidden patterns:
- Never use any type — use unknown and narrow with type guards
- Never use React.FC — use explicit function return types
- Never use index as key prop in lists unless data has no stable id
```

### <span style="color:#0D47A1;">Step 2.2: Add Inline Copilot Reminders</span>

Directly edit `.github/copilot-instructions.md` to add project-specific reminders at the top. These are the "always remember" rules:

```markdown
## Always Apply These Rules

- This is a component LIBRARY — components must have zero hardcoded
  colors or spacing; all visual values come from CSS variables or props
- Export every component from src/index.ts (the library entry point)
- Every public function and component needs a JSDoc comment
```

### <span style="color:#0D47A1;">Step 4.2: Generate useFormValidation.ts</span>

```text
@agent Implement src/hooks/useFormValidation.ts:

The hook manages form field state and validation for a set of fields.

Interface:
  useFormValidation<T extends Record<string, unknown>>(config: {
    initialValues: T,
    validators: { [K in keyof T]?: (value: T[K]) => ValidationResult }
  })

Returns:
  values: T                          — current field values
  errors: Partial<Record<keyof T, string[]>>  — current errors per field
  touched: Partial<Record<keyof T, boolean>>  — whether field was touched
  setValue(field, value): void       — update a field value
  setTouched(field): void            — mark field as touched (on blur)
  validate(): boolean                — run all validators, return isValid
  reset(): void                      — reset to initialValues, clear errors

Errors should only display for touched fields.
Import ValidationResult from @/types/index.ts
```
